# Semantic SEO And Algorithmic Authorship

Использовать для SEO/GEO-текстов РемСД: главная, страницы услуг, брендовые страницы, FAQ, описания ремонтной базы и сертификатов.

## Базовый Принцип

Страница держит один макроконтекст от H1 до последнего H2. Для РемСД макроконтекст нельзя распылять: если страница про ремонт грузовой техники, аренда идет отдельным сценарием; если страница про КАМАЗ, МАЗ или УРАЛ, остальные марки не должны перетягивать главный смысл.

## Правило Блока

Один H2 закрывает один intent cluster. После H2 сразу дать прямой ответ в 1-2 предложениях, затем добавить детали, ссылки и доказательства.

Плохо:

> Ремонтная база играет важную роль в обеспечении высокого качества сервиса.

Хорошо:

> РемСД принимает грузовые автомобили, спецтехнику, полуприцепы и тралы на ремонтной базе в Сургуте по адресу: ул. Производственная, 36.

## Entity -> Attribute -> Value

Писать так, чтобы фрагмент можно было извлечь как факт:

| Entity | Attribute | Value |
|---|---|---|
| РемСД | адрес базы | Сургут, ул. Производственная, 36 |
| РемСД | сервисный фокус | КАМАЗ, МАЗ, УРАЛ |
| ремонтная база | принимает | грузовые автомобили, спецтехнику, полуприцепы и тралы |
| сайт | документы | сертификаты, документы обучения, страницы сертификата соответствия |

Если значения нет или оно не подтверждено, не придумывать его ради убедительности.

## Passage-Level Ответы

Каждый абзац должен быть понятен отдельно от страницы. Внутри абзаца должны быть:

- сущность: РемСД, ремонтная база, сервисный центр, сертификаты, конкретный узел;
- свойство или действие: принимает, ремонтирует, подтверждает, находится, ведет;
- значение: адрес, тип техники, перечень работ, ссылка на раздел, документ.

Абзац не должен начинаться с пустого перехода: "Также", "Кроме того", "Важно отметить". Если переход нужен, добавь новую информацию, а не связку ради связки.

## Формат Под Интент

- Транзакционный интент: список услуг, карточки или матрица ссылок.
- Доверительный интент: фото базы, сертификаты, адрес, документы, видимые факты.
- Тактический интент: нумерованный процесс.
- Сравнение или характеристики: таблица `параметр -> значение`.
- FAQ: вопрос с сущностью, прямой ответ в первом предложении, затем 1-2 уточнения.

Не превращать весь сайт в таблицы. Таблица нужна там, где информация действительно структурная.

## Заголовки

Вопросительные H2 полезны для информационных страниц, но на B2B-главной не должны звучать искусственно. Если нормальный заголовок точнее, использовать нормальный:

- Хорошо: "Что закрывает ремонтная база РемСД".
- Хорошо: "Как техника попадает в работу".
- Плохо: "Когда необходимо обратиться в сервисный центр?" — если аудитория и так это знает.

## Запреты

- Не писать фейковые цены, сроки, гарантии, рейтинги, отзывы и проценты.
- Не использовать суперлативы без подтвержденного факта рядом.
- Не выдумывать "уникальные N-граммы" ради необычности. Текст должен звучать естественно по-русски.
- Не писать внутренние SEO-объяснения в публичном тексте: "чтобы не смешивать поисковые интенты" заменить на пользовательский смысл.
