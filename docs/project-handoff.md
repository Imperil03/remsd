# Project Handoff

Дата: 2026-05-09

## Что это за проект

РемСД — статический корпоративный сайт на HTML, CSS и JS для компании по ремонту грузовых автомобилей и спецтехники в Сургуте.

Главные направления:
- ремонт грузовых автомобилей и спецтехники;
- официальный сервис МАЗ, КАМАЗ, УРАЛ;
- аренда спецтехники в ХМАО как отдельный поисковый сценарий.

## Где смотреть требования

- `.agent/CONTENT_GUIDE.md` — правила текста, SEO/GEO-подача, запрет на фейковые цифры, отзывы и сроки.
- `.agent/SEO_STRUCTURE.md` — целевая структура URL.
- `docs/design-guideline.md` — текущая визуальная система и адаптивные правила.
- `AGENTS.md` — краткий входной контекст для следующей сессии.

## Текущее состояние главной

Главная страница находится в `src/pages/index.html` и собирается из partials:
- `header.html` — логотип, меню, телефон на desktop/tablet;
- `hero-home.html` — первый экран с оффером и CTA `Позвонить мастеру`;
- `official-brands.html` — три главные брендовые карточки КАМАЗ, МАЗ, УРАЛ с логотипами и вторичной сеткой обслуживаемых марок;
- `repair-services.html` — перечень услуг и строка доверия;
- `home-seo-sections.html` — порядок ремонта, симптомы обращения, зона обслуживания, FAQ, аренда и контактный CTA;
- `home-structured-data.html` — JSON-LD для главной с видимыми данными об организации, сервисе, адресе и телефоне;
- `footer.html` — футер;
- `mobile-callbar.html` — нижняя мобильная CTA-плашка.
- `base-gallery.html` — полная галерея ремонтной базы: 16 фото с превью и крупными WebP-версиями.
- `certificates-gallery.html` — 10 сертификатов и документов, включая обучение Weichai и страницы сертификата соответствия.
- `media-lightbox.html` — доступное увеличение фото и документов.
- SEO-страницы услуг генерируются из `src/data/seo-pages.json` через шаблон `src/templates/seo-page.html`.
- `/remontnaya-baza/` и `/sertifikaty/` лежат отдельными HTML-страницами в `src/pages/`, поэтому генератор SEO-страниц их не перетирает.
- CSS/JS подключаются с версией `assetVersion` из `tools/build.js`, чтобы GitHub Pages и браузер не держали старые стили.
- Фото и логотипы готовятся скриптом `npm.cmd run prepare:media`, который скачивает исходники с remsd.ru, режет их под нужные размеры и сохраняет WebP без метаданных.

## Принятые дизайн-решения

- Hero оставлен легким: один H1, короткое описание и одна кнопка звонка.
- Блок официального сервиса КАМАЗ/МАЗ/УРАЛ стоит сразу после hero, потому что это первый коммерческий слой.
- Блок `Что нужно сделать с техникой?` удален как дублирующий: брендовые карточки закрывают выбор марки, а перечень услуг закрывает выбор работ.
- Подробный перечень услуг идет сразу после брендовых карточек. Ссылки внутри списков ведут только на существующие или укрупненные страницы, например `/remont-gbc/`.
- Ниже медиа-доказательств добавлены блоки для SEO и нейроответов: как проходит ремонт, когда обращаться в сервис, зона обслуживания, FAQ, аренда как отдельный сценарий и контактный CTA.
- JSON-LD на главной описывает только видимые факты: РемСД, телефон, адрес базы, Сургут/ХМАО и главную страницу. Цены, рейтинги, отзывы и сроки не добавлены.
- На мобильных ширинах телефон убран из шапки: вместо него нижняя фиксированная кнопка `Позвонить`, которая появляется после скролла.
- В карточках услуг пока не используются изображения: реальные фото вынесены в отдельный блок ремонтной базы ниже услуг, чтобы не перегружать навигационные карточки.
- Галерея сделана без слайдера: крупная адаптивная сетка открывает фото и документы в доступном lightbox с клавиатурой.
- Локальные копии agent skills удалены из `.agent/skills`; в репозитории остаются только проектные инструкции.

## Как продолжить с другого компьютера

```bash
git clone https://github.com/Imperil03/remsd.git
cd remsd
node tools/build.js
```

Открыть готовую главную:

```text
dist/index.html
```

В PowerShell можно использовать:

```bash
npm.cmd run build
```

Продакшен-превью:

```text
https://imperil03.github.io/remsd/
```

## Следующие шаги

1. Добавить блок формы заявки ниже услуг или после блока доверия.
2. Расширить раздел аренды спецтехники отдельными страницами из `.agent/SEO_STRUCTURE.md`, не смешивая его с ремонтом.
3. Проверить и дополнить страницы КАМАЗ, МАЗ, УРАЛ фактическими подтверждениями официального сервиса: сертификаты, документы, фото, реквизиты.
4. После появления дополнительного фактического контента проверить title/description, внутренние ссылки и микроразметку.

## Проверки перед коммитом

```bash
node tools/build.js
git diff --check
```

Для визуальной проверки можно поднять локальный сервер из `dist/`:

```bash
python -m http.server 4173 --directory dist --bind 127.0.0.1
```

И сделать скриншот через Playwright, не добавляя его в зависимости проекта:

```bash
npx.cmd --yes playwright screenshot --viewport-size=390,844 http://127.0.0.1:4173/ mobile.png
```
