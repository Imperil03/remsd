# Project Handoff

Дата: 2026-05-11

## Что это за проект

РемСД — статический корпоративный сайт на HTML, CSS и JS для компании по ремонту грузовых автомобилей и спецтехники в Сургуте.

Главные направления:
- ремонт грузовых автомобилей и спецтехники;
- официальный сервис МАЗ, КАМАЗ, УРАЛ;
- аренда спецтехники в ХМАО как отдельный поисковый сценарий.

## Где смотреть требования

- `.agent/CONTENT_GUIDE.md` — правила текста, SEO/GEO-подача, запрет на фейковые цифры, отзывы и сроки.
- `.agent/SEO_STRUCTURE.md` — целевая структура URL.
- `.agent/CLAIMS_LEDGER.md` — подтвержденные публичные факты: телефон, адрес, документы, услуги и запреты.
- `.agent/agents/` — инструкции для текстового автора и ревьюера.
- `docs/design-guideline.md` — текущая визуальная система и адаптивные правила.
- `AGENTS.md` — краткий входной контекст для следующей сессии.

## Текущее состояние главной

Главная страница находится в `src/pages/index.html` и собирается из partials:
- `header.html` — логотип, меню, телефон на desktop/tablet;
- `hero-home.html` — первый экран с оффером и CTA `Позвонить мастеру`;
- `official-brands.html` — компактный proof-блок официального сервиса КАМАЗ, МАЗ, УРАЛ с логотипами и вторичной сеткой обслуживаемых марок;
- `repair-services.html` — основной коммерческий блок главной: ремонт грузовых автомобилей и спецтехники по узлам;
- `home-process.html` — industrial timeline приема техники в ремонт;
- `company-about.html` — паспорт сервиса с адресом, телефоном, статусом, типами техники и документами;
- `home-seo-sections.html` — service map ремонтной базы, зона обслуживания, FAQ, аренда и контактный CTA;
- `home-structured-data.html` — JSON-LD для главной с видимыми данными об организации, сервисе, адресе и телефоне;
- `footer.html` — футер;
- `mobile-callbar.html` — нижняя мобильная CTA-плашка.
- `base-gallery.html` — полная галерея ремонтной базы: 16 фото с превью и крупными WebP-версиями.
- `certificates-gallery.html` — 10 сертификатов и документов, включая обучение Weichai и страницы сертификата соответствия.
- `media-lightbox.html` — доступное увеличение фото и документов.
- SEO-страницы услуг генерируются из `src/data/seo-pages.json` через шаблон `src/templates/seo-page.html`.
- `/remontnaya-baza/` и `/sertifikaty/` лежат отдельными HTML-страницами в `src/pages/`, поэтому генератор SEO-страниц их не перетирает.
- CSS/JS подключаются с версией `assetVersion` из `tools/build.js`, чтобы GitHub Pages и браузер не держали старые стили.
- Фото и логотипы готовятся скриптом `npm.cmd run prepare:media`, который скачивает исходники с remsd.ru, режет их под нужные размеры и сохраняет WebP без метаданных.

## Принятые дизайн-решения

- Hero оставлен легким: один H1, короткое описание и одна кнопка звонка.
- Из временной версии `/v2/` перенесены только удачные UI-решения: более плотная типографика, стиль брендовых карточек, механика блока приема в ремонт и идея карты в географии. Неподтвержденные обещания из v2 не переносились.
- Типографика главной переведена на локальные `woff2`: Montserrat для заголовков/интерфейсных акцентов и Source Sans 3 для основного текста.
- Главная v1 продвигает общий интент: ремонт грузовых автомобилей и спецтехники в Сургуте. Официальный сервис КАМАЗ/МАЗ/УРАЛ стоит сразу после hero как proof-сигнал и навигация в брендовые разделы.
- После официальных брендов идет основной перечень услуг по узлам, затем процесс обращения и паспорт сервиса.
- Блок `Что нужно сделать с техникой?` удален как дублирующий: брендовые карточки закрывают выбор марки, а перечень услуг закрывает выбор работ.
- Подробный перечень услуг идет сразу после брендовых карточек. Ссылки внутри списков ведут только на существующие или укрупненные страницы, например `/remont-gbc/`.
- Ниже услуг добавлены блоки для SEO и нейроответов: как техника попадает в работу, какие работы выполняет ремонтная база, география работ с Яндекс.Картой и fallback-ссылкой, FAQ, аренда как отдельный сценарий и контактный CTA.
- Для блоков доверия, процесса и базы выбран стиль `Industrial Editorial`: отдельные proof-элементы вместо табличной плашки, маршрут ремонта в виде технической линии и карта работ базы с темной паспортной панелью.
- На главной избегаем одинаковых карточных сеток подряд: брендовые карточки, перечень услуг, галерея, proof-rail, timeline и service map должны отличаться по ритму и назначению.
- JSON-LD на главной описывает только видимые факты: РемСД, телефон, адрес базы, Сургут/ХМАО и главную страницу. Цены, рейтинги, отзывы и сроки не добавлены.
- На мобильных ширинах телефон убран из шапки: вместо него нижняя фиксированная кнопка `Позвонить`, которая появляется после скролла.
- В карточках услуг пока не используются изображения: реальные фото вынесены в отдельный блок ремонтной базы ниже услуг, чтобы не перегружать навигационные карточки.
- Галерея сделана без слайдера: крупная адаптивная сетка открывает фото и документы в доступном lightbox с клавиатурой.
- Project skills теперь версионируются в `.agent/skills/`; состав, лицензии и правила подключения описаны в `docs/skills.md`.

## Как продолжить с другого компьютера

```bash
git clone https://github.com/Imperil03/remsd.git
cd remsd
node tools/build.js
```

Открыть готовую главную:

```text
dist/index.html
```

В PowerShell можно использовать:

```bash
npm.cmd run build
```

Продакшен-превью:

```text
https://imperil03.github.io/remsd/
```

## Следующие шаги

1. Посмотреть прод-версию свежим взглядом после деплоя: первый экран, услуги, база, сертификаты, mobile CTA и контактный сценарий.
2. Добавить блок формы заявки ниже услуг или после доказательств доверия, если он не будет перегружать главную.
3. Расширить раздел аренды спецтехники отдельными страницами из `.agent/SEO_STRUCTURE.md`, не смешивая его с ремонтом.
4. Проверить и дополнить страницы КАМАЗ, МАЗ, УРАЛ фактическими подтверждениями официального сервиса: сертификаты, документы, фото, реквизиты.
5. После появления дополнительного фактического контента проверить title/description, внутренние ссылки и микроразметку.

## Проверки перед коммитом

```bash
node tools/build.js
git diff --check
```

Для визуальной проверки можно поднять локальный сервер из `dist/`:

```bash
python -m http.server 4173 --directory dist --bind 127.0.0.1
```

И сделать скриншот через Playwright, не добавляя его в зависимости проекта:

```bash
npx.cmd --yes playwright screenshot --viewport-size=390,844 http://127.0.0.1:4173/ mobile.png
```

## Project Skills

Project skills версионируются в `.agent/skills/` и документированы в `docs/skills.md`.

Перед работой с дизайном или текстами читать:

1. `.agent/CONTENT_GUIDE.md`
2. `.agent/SEO_STRUCTURE.md`
3. `docs/skills.md`
4. Релевантный skill из `.agent/skills/`

В репозиторий добавлены только выбранные skills с понятной релевантностью и лицензией. Внешние репозитории без `LICENSE` описаны в `docs/skills.md`, но не скопированы в публичный репозиторий.
