# AGENTS.md

## Проект

РемСД — статический сайт сервисного центра по ремонту грузовых автомобилей и спецтехники в Сургуте.

Рабочий продакшен-превью:
- GitHub Pages: https://imperil03.github.io/remsd/

Внешний домен компании:
- https://remsd.ru/

## Главный SEO-фокус

1. Официальный сервис и ремонт КАМАЗ в Сургуте.
2. Официальный сервис и ремонт МАЗ в Сургуте.
3. Официальный сервис и ремонт УРАЛ в Сургуте.
4. Ремонт агрегатов грузовых автомобилей и спецтехники: двигатель, ГБЦ, КПП, ходовая, тормоза, гидравлика, электрика, топливная система, охлаждение, SCR/AdBlue, ТО.
5. Аренда спецтехники — отдельное направление, не смешивать с ремонтом.

## Где лежит контекст

- `.agent/SEO_STRUCTURE.md` — актуальная SEO-архитектура и запрет на тонкие страницы "услуга × марка".
- `.agent/CONTENT_GUIDE.md` — правила публичного текста.
- `docs/project-handoff.md` — состояние проекта, что уже сделано и как продолжать.
- `docs/design-guideline.md` — визуальные правила.
- `src/data/seo-pages.json` — контент генерируемых страниц.

## Как работать

- Сначала читать `.agent/SEO_STRUCTURE.md` и `docs/project-handoff.md`.
- Не добавлять отдельные страницы вида `/remont-gbc-kamaz/`, `/remont-kpp-maz/`, `/remont-elektriki-ural/` без отдельного решения.
- Для новых SEO-страниц сначала добавлять объект в `src/data/seo-pages.json`, затем запускать `node tools/build.js`.
- После изменений проверять:

```bash
node tools/build.js
git diff --check
```

## Деплой

`dist/` отслеживается в git и используется GitHub Pages workflow. После сборки нужно коммитить и исходники, и обновленный `dist/`.
